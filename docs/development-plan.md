# План разработки

## Правила работы

- Сохранять изменения небольшими и удобными для проверки.
- Останавливаться на каждой контрольной точке до начала следующего шага.
- Не добавлять несвязанные продуктовые функции и декоративные абстракции.
- Не создавать commits, не выполнять push и не создавать удалённый репозиторий без явного разрешения.
- Не хранить секреты в исходном коде и использовать `.env.example` только как шаблон.

- [x] Добавить CORS allowlist и применить настроенную proxy policy.
- [x] Добавить общий error mapper для ответов `400`, `401`, `409`, `429`, `503` и `500`.
- [x] Проверить повторный и конкурентный refresh, runtime dependency failures и redaction логов.
- [x] Запрещать запуск integration tests, если URL не указывают на тестовые ресурсы.

Контрольная точка: из конкурентных refresh проходит ровно один, старые сессии нельзя использовать
повторно, защищённый запрос проверяет существование пользователя, а чувствительные значения не
попадают в логи.
